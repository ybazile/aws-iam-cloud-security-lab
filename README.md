In this AWS IAM project, I simulated working as a DevOps engineer at Nike responsible for managing cloud security and access control.

The project focused on launching EC2 instances, creating IAM policies, configuring IAM users and groups, and enforcing least-privilege access between development and production environments.

I also validated permissions using the AWS IAM Policy Simulator to ensure secure access controls were working correctly.

## Technologies Used

- AWS EC2
- AWS IAM
- IAM Policies
- IAM Users & Groups
- IAM Policy Simulator
- JSON Policies
- Access Control
- Cloud Security

## Key Concepts Learned

- Principle of Least Privilege
- IAM Policy Structure
- Resource-Based Permissions
- Environment Segmentation
- Access Management
- Explicit Deny vs Allow
- EC2 Instance Management
- Policy Simulation & Validation

## Project Highlights

- Launched and configured AWS EC2 instances for production and development environments
- Implemented IAM policies using JSON to enforce least-privilege access controls
- Configured IAM users and groups to manage role-based access permissions
- Restricted EC2 actions based on resource tags using conditional IAM policies
- Prevented privilege escalation by explicitly denying tag modification actions
- Validated IAM permissions using AWS IAM Policy Simulator
- Tested environment-specific access controls by logging in as restricted IAM users
